# Requirements — Email Compose (Phase 3 completion)

## Scope

This feature closes the two open items in Phase 3:

1. **Admin compose UI** — a subject + body form on the admin subscribers page that calls `POST /api/send-email`.
2. **Welcome email** — a branded email sent automatically when a new subscriber signs up via `POST /api/subscribe`.

Everything else in the email system (Resend SDK, unsubscribe flow, subscriber CRUD) is already implemented and is out of scope for this work.

---

## Locked decisions

### Blast scope — all subscribers, no segmentation
`POST /api/send-email` sends to every row in the `subscribers` table. There are no categories, opt-in topics, or audience filters in this phase. If segmentation is ever needed, it is a Phase 6 item.

### HTML-only emails
All outgoing emails are HTML-only. No plain-text `text:` field is passed to Resend. This matches the existing templates in `src/lib/resend.ts` and is acceptable given the simple formatting of PAC communications.

### Token-based unsubscribe — no changes
The unsubscribe mechanism (`base64url(email)` token, `GET /api/unsubscribe?token=`) is already in place and is not changed by this work. Every outgoing email (blast and welcome) must include the unsubscribe URL generated by `generateUnsubscribeUrl()`.

### Admin-only blast trigger
Only an authenticated admin session can call `POST /api/send-email`. The route already enforces `isAuthenticated()`. There is no public or API-key-only path for triggering blasts. The compose form lives exclusively inside the `/admin` area.

---

## Constraints

- The welcome email send must be **non-blocking**: if Resend fails to deliver the welcome email, `POST /api/subscribe` still returns `{ success: true }`. Log the error server-side; do not surface it to the subscriber.
- The compose form must **disable the send button** while the request is in flight to prevent duplicate sends.
- The `pacName` in email templates comes from `getSchoolSettings()` (Supabase `settings` table) — do not hardcode it.
- Do not introduce new environment variables. All required variables (`RESEND_API_KEY`, `RESEND_FROM_EMAIL`) are already defined.
